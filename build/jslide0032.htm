Binary data

When integrating binary files like the photo collections of several archived conference years, it makes sense to store them in one central place. That's what the bindata directory under build is intended for.

To use this directory, just copy all those files into it and refer to them by URL's: 


  \URL{a="bindata/proceedings-2003.pdf"}<PDF file>

Please note:

    This technique neither works nor is necessary for images integrated via \IMAGE tags.
    This approach applies to CD sources only. Stuff provided by speakers can be stored unmodified in the talk directories as described.
